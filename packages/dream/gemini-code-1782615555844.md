# CX_lab Dojo: Antigravity Self-Improvement Specification

This specification defines the runtime boundaries, behavioral criteria, and data contracts for the `antigravity-preview-05-2026` Managed Agent loop. This file serves as the system memory anchor during offline **Loop 3: Cross-Call Dreaming** passes.

---

## 1. System Positioning & Objective
* **Core Goal:** Classify a customer's profile, predict next-turn intent, execute localized playbooks, identify prediction/containment failure patterns, and run offline dream optimization steps to upgrade playbooks without breaking execution loops.
* **Core Loop Architecture:**
  1. **Loop 1 (Live Execution):** Real-time intent tracking and next-turn intent probability matrix distribution.
  2. **Loop 2 (Offline Self-Play):** Synthetic call generation to test out boundary scenarios.
  3. **Loop 3 (Cross-Call Dreaming):** Vector-indexed review of failure points to generate text policy patches.

## 3. The Three-Source Intent Prediction Engine
The target optimization variable for the agent is $P_{final}(next\_intent)$, calculated dynamically via:

$$P_{final}(next\_intent) = \alpha \cdot P_{model} + \beta \cdot P_{profile} + \gamma \cdot P_{intent\_transition}$$

### Adaptive Weight Shifts
The agent shifts its structural confidence layout once it acquires collective historical training tokens:
* **Generation 1 (Baseline):** $\alpha = 0.60, \beta = 0.25, \gamma = 0.15$
* **Generation 3 (Dream Optimized):** $\alpha = 0.45, \beta = 0.25, \gamma = 0.30$ (Trusts learned structural transition histories higher).

---

## 4. Evaluation Rubric & Reward Function

The optimization engine scores each session output array (`ConversationResult`) against a composite reward function:

$$R = 0.30 \cdot \text{containment} + 0.20 \cdot \text{resolution} + 0.15 \cdot \text{sentiment\_recovery} + 0.15 \cdot \text{compliance} + 0.10 \cdot \text{prediction\_quality} + 0.05 \cdot \text{efficiency} + 0.05 \cdot \text{retention} - \text{penalties}$$

### Pruning Vectors
* **Hard Prune Triggers (Immediate Demotion):** Hallucinated refund policies, toxic interactions, compliance/privacy violations, unsupported inventory/order updates.
* **Soft Prune Triggers (Weight Reduction):** Negative sentiment trajectory over three turns, reward score dropping below target thresholds, high predictive miss rates correlated with human agent handoffs.

## 5. Dream Pass Input & Policy Diff Contract

When similarities match via `pgvector` lookups on DigitalOcean, the Antigravity agent executes a **Playbook Patch Compilation**.

### Target Sample Scenario: Maya (Late Birthday Gift)
* **Failure Pattern:** `Loyal Shopper` + `Urgent` + `Late Delivery` + `policy_first_response` $\rightarrow$ triggers a text escalation and cancel/refund threat due to emotional friction.
* **Optimization Goal:** Generate a policy markdown diff that shifts the playbook structure to `inventory_lookup_first` then `expedited_replacement_offer`.

### Guardrail Anchor
> "The agent does not rewrite itself during a customer call. It captures evidence. Then the dojo dreams over transcripts, verifies repeated misses, and promotes safer policies for the next generation."
